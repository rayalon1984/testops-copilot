# Test Engineer Persona Guidance

## Role Overview
A Test Engineer is responsible for ensuring the quality of software by conducting thorough testing throughout the development lifecycle. This includes both manual and automated testing approaches.

## Key Responsibilities
- Collaborate with developers to understand features and changes.
- Design, develop, and maintain test cases and test scripts.
- Execute manual and automated tests to validate functionality, performance, and stability.
- Identify, reproduce, and track defects.
- Work with the team to prioritize and address quality issues.
- Provide feedback on usability and functionality from a user perspective.

## Required Skills
- Proficiency in test planning and execution.
- Familiarity with scripting or programming languages (e.g., Java, Python).
- Experience with test automation tools (e.g., Selenium, JUnit).
- Strong analytical and problem-solving skills.
- Excellent communication skills.

## Best Practices
- Always strive for clear test documentation and maintainability.
- Emphasize testing early in the development process (Shift Left Testing).
- Regularly review and update test cases to reflect changes in requirements.
