# 🚀 Local Development Setup - TestOps Companion

## Quick Start (3 Steps - 5 Minutes)

### Prerequisites
1. **Docker Desktop** - [Download](https://www.docker.com/products/docker-desktop) and make sure it's running
2. **Node.js 18+** - [Download](https://nodejs.org/)

### Step 1: Install Dependencies
```bash
npm install
cd backend && npm install
cd ../frontend && npm install
cd ..
```

### Step 2: Set Up Local Environment
```bash
# This automatically:
# - Starts PostgreSQL, Redis, Weaviate in Docker
# - Sets up database schema
# - Waits for everything to be ready
npm run local:setup
```

**During setup, you'll be asked for:**
- GitHub token (optional - for GitHub integrations)
- Jira API token (optional - for Jira integrations)
- Jira base URL (optional)
- Jira project key (optional)

*You can skip these by pressing Enter - they're only needed for specific features*

### Step 3: Start Development Servers
```bash
# Start both frontend and backend
npm run dev
```

**That's it!** 🎉

- Frontend: http://localhost:5173
- Backend: http://localhost:4000
- Database Admin (Adminer): http://localhost:8080
  - System: PostgreSQL
  - Server: db
  - Username: postgres
  - Password: postgres
  - Database: testops

## 📋 Common Commands

### Development
```bash
npm run dev              # Start both frontend & backend
npm run dev:backend      # Start backend only
npm run dev:frontend     # Start frontend only
```

### Docker Services
```bash
npm run local:start      # Start Docker services (DB, Redis, Weaviate)
npm run local:stop       # Stop Docker services
docker-compose ps        # Check service status
```

### Database
```bash
cd backend
npm run db:generate      # Generate Prisma client
npm run db:migrate       # Run migrations
npm run db:studio        # Open Prisma Studio (DB GUI)
npm run db:reset         # Reset database (⚠️  deletes all data)
```

### Testing & Quality
```bash
npm run test             # Run all tests
npm run lint             # Lint code
npm run typecheck        # Type check TypeScript
npm run build            # Build for production
```

## 🔧 Troubleshooting

### "Can't reach database server"
**Solution:**
```bash
# Make sure Docker is running
docker ps

# Restart services
npm run local:stop
npm run local:setup
```

### "Port already in use"
**Solution:**
```bash
# Check what's using the ports
lsof -i :5173  # Frontend
lsof -i :4000  # Backend
lsof -i :5432  # PostgreSQL

# Kill the process or change ports in .env files
```

### "Docker command not found"
**Solution:**
1. Install Docker Desktop: https://www.docker.com/products/docker-desktop
2. Start Docker Desktop
3. Try again

### Fresh Start
```bash
# Nuclear option - start completely fresh
npm run local:stop
rm -rf node_modules backend/node_modules frontend/node_modules
rm backend/.env frontend/.env
npm run local:setup
```

## 🐳 Docker Commands Reference

```bash
# View logs
docker-compose logs -f                # All services
docker-compose logs -f db            # PostgreSQL only
docker-compose logs -f backend       # Backend only

# Restart a service
docker-compose restart db
docker-compose restart backend

# Access PostgreSQL directly
docker-compose exec db psql -U postgres -d testops

# Access Redis CLI
docker-compose exec redis redis-cli

# View running containers
docker-compose ps

# Stop everything
docker-compose down

# Stop and remove all data
docker-compose down -v
```

## 💡 Development Tips

### Hot Reload
Both frontend and backend support hot reload:
- Frontend: Vite auto-reloads on file changes
- Backend: Nodemon restarts on file changes

### Database Changes
```bash
# After modifying prisma/schema.prisma:
cd backend
npm run db:generate          # Update Prisma client
npx prisma migrate dev       # Create migration
```

### Viewing Data
1. **Adminer** (Web UI): http://localhost:8080
2. **Prisma Studio**: `cd backend && npm run db:studio`
3. **Direct SQL**: `docker-compose exec db psql -U postgres -d testops`

### API Testing
```bash
# Test backend health
curl http://localhost:4000/health

# Test database connection
curl http://localhost:4000/api/v1/metrics/health
```
