# Quick Start Guide

Get started with TestOps Companion in minutes. Choose the path that fits your needs.

---

## Path 1: Demo Mode (Fastest — No Docker)

**Best for**: Evaluation, demos, UI development, screenshots

**Prerequisites**: Node.js 18+ and npm 9+ only

```bash
# Clone and start
git clone https://github.com/rayalon1984/testops-companion.git
cd testops-companion
npm install
npm run dev:simple
```

**What happens:**
1. Installs all dependencies (root, backend, frontend)
2. Generates SQLite Prisma client
3. Seeds 1,600+ test failures, 150 test runs, 15 pipelines
4. Starts backend on http://localhost:3000 and frontend on http://localhost:5173
5. Opens browser automatically

**Login** — all accounts use password `demo123`:
- `admin@testops.ai` — Site Admin
- `lead@testops.ai` — QA Lead
- `engineer@testops.ai` — QA Engineer
- `viewer@testops.ai` — Stakeholder

> AI copilot runs in mock mode with realistic demo data. All 18 tools and persona routing work without API keys.

---

## Path 2: Development Mode (Docker + Local Servers)

**Best for**: Backend development, real data, full integration testing

**Prerequisites**: Node.js 18+, npm 9+, Docker Desktop

```bash
git clone https://github.com/rayalon1984/testops-companion.git
cd testops-companion
npm install

# Start infrastructure (PostgreSQL, Redis, Weaviate)
npm run local:start

# Start application servers
npm run dev
```

**URLs:**
- Frontend: http://localhost:5173
- Backend API: http://localhost:3000
- Adminer (Database UI): http://localhost:8080

### Environment Configuration

The setup creates `.env` files from templates. Key settings in `backend/.env`:

```env
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/testops
REDIS_URL=redis://localhost:6379
JWT_SECRET=<auto-generated>
CORS_ORIGIN=http://localhost:5173
```

And `frontend/.env`:

```env
VITE_API_URL=http://localhost:3000
VITE_WEBSOCKET_URL=ws://localhost:3000
```

Full templates: `backend/.env.example` and `frontend/.env.example`.

## Path 3: Production Deployment (Docker Only)

**Best for**: Staging/production, beta users

See the **[Production Quickstart](PRODUCTION_QUICKSTART.md)** for full instructions. The short version:

```bash
cp .env.production.example .env.production
# Edit .env.production: set DATABASE_URL, JWT_SECRET, JWT_REFRESH_SECRET

docker compose -f docker-compose.ghcr.yml up -d
# Dashboard: http://localhost
```

## First Steps After Setup

### 1. Login (Demo Mode)

All accounts use password `demo123`:
- `admin@testops.ai` — Site Admin
- `lead@testops.ai` — QA Lead
- `engineer@testops.ai` — QA Engineer

All features are pre-loaded.

### 2. Register an Account (Development/Production Mode)

```bash
curl -X POST http://localhost:3000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "SecurePass123!",
    "firstName": "Admin",
    "lastName": "User"
  }'
```

### 3. Configure Integrations (Optional)

Add to `backend/.env` as needed:

```env
# Jira
JIRA_BASE_URL=https://your-domain.atlassian.net
JIRA_API_TOKEN=your-api-token

# GitHub
GITHUB_TOKEN=ghp_your_personal_access_token

# Slack
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/YOUR/WEBHOOK/URL
```

Restart the backend after adding integration keys.

## Common Issues

### "Prisma client not found" (Demo Mode)

```bash
cd backend && npx prisma generate
```

### "Database is locked" (Demo Mode)

```bash
rm backend/prisma/dev.db
npm run dev:simple
```

### "Can't reach database" (Development Mode)

```bash
docker ps                  # Verify containers are running
docker compose logs db     # Check PostgreSQL logs
npm run local:stop && npm run local:start  # Restart services
```

### Port 3000 Already in Use

1. Edit `backend/.env`: set `PORT=3001`
2. Edit `frontend/.env`: set `VITE_API_URL=http://localhost:3001`
3. Restart: `npm run dev`

### Port 5173 Already in Use

Vite auto-increments (tries 5174, 5175, etc.). Update `backend/.env` if needed:

```env
CORS_ORIGIN=http://localhost:5174
```

### Authentication Errors

Verify these are set correctly in `backend/.env`:

```env
JWT_SECRET=<any-string-at-least-32-chars>
JWT_EXPIRES_IN=24h
JWT_REFRESH_SECRET=<different-string-at-least-32-chars>
JWT_REFRESH_EXPIRES_IN=7d
```

### Database Reset (Development Mode)

```bash
# Safe reset — preserves migration history
cd backend && npx prisma migrate reset --force
```

> **Warning**: Never delete `backend/prisma/migrations/` manually. The migration baseline is required for production deployments.
